# 图形用户界面的待办事项管理器（Web 版）

一个基于“本地 Python 服务 + 浏览器界面”的待办事项管理器。  
当前版本将任务数据保存到本机 `data/web_store/` 下的 JSON 文件中，而不是仅保存在浏览器 `localStorage` 中，因此同一台电脑上更换浏览器访问时，仍可读取同一账户的数据。

## 项目概览

这个项目实现了一个带图形界面的待办系统，重点不是单纯的任务增删改查，而是：

- 多账户登录与独立数据存储
- 父任务 / 子任务的层级管理
- 基于 DDL 的排序、颜色提示与超期状态
- 列表内展开详情，而不是弹出新页面
- DDL 合法性校验与到期提醒
- 本地文件持久化

页面布局为横版：

- 上方是任务列表
- 下方是任务输入区
- 输入区会根据当前模式切换状态色
- 支持创建任务、创建子任务、修改任务、修改子任务四种模式

## 功能说明

### 1. 账户系统

- 支持注册、登录、退出登录、切换账号
- 每个账号的数据彼此独立
- 登录后自动从本地 JSON 文件加载该用户的任务数据
- 浏览器仅使用 `localStorage` 记录上次登录的用户名，用于恢复会话

### 2. 任务管理

- 支持创建、修改、删除父任务
- 支持创建、修改、删除子任务
- 仅支持一层子任务，不支持子任务继续嵌套子任务
- 当某个父任务的所有子任务全部完成时，父任务会自动完成
- 有子任务的父任务不能直接手动标记完成

### 3. 任务展示与交互

- 任务按规则自动排序
- 悬停任务卡片时显示操作按钮
- 点击任务后，详情直接在列表中展开
- 详情中显示 DDL、任务内容，以及子任务列表
- 子任务列表默认收起，并显示“已完成 n/m”
- 展开后可继续查看子任务详情，并支持收起

### 4. DDL 与状态规则

- DDL 必须晚于当前系统时间
- 子任务 DDL 必须早于父任务 DDL
- 修改父任务时，父任务 DDL 必须晚于其全部子任务 DDL
- 如果父任务存在未完成子任务，列表中的剩余时间以最近的未完成子任务 DDL 为准

### 5. 排序与颜色语义

- 已超期未完成任务：置顶显示，标记为蓝色
- 未完成任务：按 DDL 从近到远排序
- 已完成任务：置底显示，标记为灰色
- 紧急程度根据距离 DDL 的时间变化，使用红 / 黄 / 绿等颜色区分

### 6. 提醒机制

- 在登录状态下运行时，未完成任务会在 DDL 前 5 分钟提醒一次
- 到达 DDL 时再次提醒一次
- 提醒以页面弹窗形式展示

## 技术结构

### 前端

- `index.html`：页面结构
- `styles.css`：界面样式
- `app.js`：前端交互逻辑、任务渲染、DDL 校验、提醒逻辑

### 后端

- `server.py`：本地 HTTP 服务，负责静态页面服务与任务数据读写

### 数据存储

- `data/web_store/accounts.json`：账户信息
- `data/web_store/tasks_<hash>.json`：某个账户对应的任务数据

## 运行环境

- Python 3.10+
- 任意现代浏览器（Chrome / Edge / Firefox / Safari）
- 页面字体使用 Google Fonts：`Manrope`、`Noto Sans SC`
- 若无法访问特定字体，浏览器会自动回退到本地可用字体，不影响功能。
当前 Web 版运行 `server.py` 时仅依赖 Python 标准库，无需额外安装第三方后端依赖。

## 运行方式

在项目根目录启动本地服务：

```bash
python server.py
```

然后在浏览器中访问：

```text
http://127.0.0.1:8000
```

如果需要修改端口，可以设置环境变量 `FLOWDO_PORT` 后再启动。

## 使用流程

1. 启动 `server.py`
2. 打开浏览器访问本地地址
3. 注册账号或直接登录已有账号
4. 在底部输入区填写任务主题、DDL 和任务内容
5. 在任务列表中展开详情、修改任务、添加子任务或完成任务

## 项目目录

```text
.
├─ index.html              # 页面结构
├─ styles.css              # 页面样式
├─ app.js                  # 前端逻辑
├─ server.py               # 本地服务
├─ README.md               # 项目说明
├─ DEPENDENCIES.md         # 依赖说明
├─ requirements.txt        # 依赖记录文件
└─ data/
   └─ web_store/
      ├─ accounts.json
      └─ tasks_<hash>.json
```

## 说明

- 当前仓库主要提供的是 Web 版实现
- 任务数据保存在本地文件中，因此删除 `data/web_store/` 下的文件会清空对应数据
- 如果本地服务未启动，前端会提示无法连接数据服务
